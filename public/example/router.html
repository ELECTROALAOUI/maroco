Router Router

            seajs.use(['ui', 'ui/router'], function(UI, router) {
                //IE67当第一次调用router的时候
                router({
                    //以#开头的属性会和hash去匹配
                    '#layout/:id': "method",
                    "#*": function(mod) {
                        //从上到下匹配，取第一个执行，没有则不执行
                    },
                    //自定义属性，不参加匹配
                    method:function(){}
                })
            });
缺点:
无法使用锚点定位功能
hash无法传递大量数据
不可控，无法确定页面上不会出现不受控制的链接来修改hash
优点:
浏览器产生历史记录，方便回退

综上所述，hashchange这种东西正式项目不要使用，一般是html5的history配合后端无缝跳转才有意义。
